# Phase 3.5b — Realtime Client (Apollo + SSE) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the frontend consume the backend's GraphQL SSE subscriptions for the first time — a live notification bell badge and live comment appends — by adding a delta-only Apollo Client over `graphql-sse`. Add a per-user `notificationAdded` subscription on the backend.

**Architecture:** The existing SSR + server-action data path is the **single source of truth**; Apollo is added **only** for live deltas. A client-only `ApolloProvider` mounts high in the tree and is used solely for subscriptions (and a couple of live queries). The browser authenticates the SSE connection by calling a server action that returns the current access token (httpOnly cookie `pf_at`), passed as a `Bearer` header on each (re)connect — `graphql-sse` uses `fetch`, so custom headers work, and CORS already allows `Authorization` + credentials from the web origin. Live deltas mutate scoped local component state, never a parallel full fetch. Board/list `taskUpdated` subscription is **deferred** (per spec §9).

**Tech Stack:** GraphQL Yoga SSE subscriptions (existing), `graphql-yoga` `createPubSub` per-user topics, Pothos; frontend `@apollo/client` + `graphql-sse` + `graphql`, Next.js 16 (App Router), React 19.

> **Prerequisites:** Slice 3.5a merged (the `MENTION`/`COMMENT_ASSIGNED` notifications and `notificationService.notify` it builds on) and the i18n foundation (Tasks 1–6 of the i18n plan) for any new UI strings.

```typescript
import { GraphQLError } from 'graphql';
import { pubsub } from '../pubsub.js';
import type { GQLContext } from '../context.js';

/**
 * Binds the subscription to the AUTHENTICATED user's id from context — never
 * the client-supplied arg. A user can only receive their own notifications.

export function notificationAddedSubscribe(_root: unknown, _args: { userId?: string | null }, ctx: GQLContext) {
  if (!ctx.user) throw new GraphQLError('Unauthorized', { extensions: { code: 'UNAUTHENTICATED' } });

}
```

```typescript
'use server';

import { cookies } from 'next/headers';
import { COOKIE } from '../cookies';

/**
 * Returns the current access token for bootstrapping the browser's SSE
 * connection. The token lives in an httpOnly cookie that client JS cannot read,
 * so the client calls this on each (re)connect to obtain a fresh value.
 * Returns null when there is no session (the SSE link then skips connecting).
 */
export async function getRealtimeToken(): Promise<{ token: string } | null> {
  const token = (await cookies()).get(COOKIE.access)?.value;
  return token ? { token } : null;
}
```

- [ ] **Step 1: Create the SSE ApolloLink adapter**

Create `apps/next-web/src/lib/realtime/sseLink.ts`:

```typescript
'use client';

import { ApolloLink, Observable, type FetchResult, type Operation } from '@apollo/client/core';
import { type Client, type ClientOptions, createClient } from 'graphql-sse';
import { print } from 'graphql';

/** Wraps a graphql-sse client as an ApolloLink (subscriptions + queries over SSE). */
export class SSELink extends ApolloLink {
  private client: Client;
  constructor(options: ClientOptions) {
    super();
    this.client = createClient(options);
  }
  request(operation: Operation): Observable<FetchResult> {
    return new Observable((sink) => {
      return this.client.subscribe<FetchResult>(
        { ...operation, query: print(operation.query) },
        {
          next: sink.next.bind(sink),
          complete: sink.complete.bind(sink),
          error: sink.error.bind(sink),
        },
      );
    });
  }
}
```

> Import paths may differ by Apollo major (`@apollo/client/core` vs `@apollo/client`). Use what the installed version exports; the adapter logic is identical.

- [ ] **Step 2: Create the client factory**

Create `apps/next-web/src/lib/realtime/apolloClient.ts`:

```typescript
'use client';

import { ApolloClient, InMemoryCache } from '@apollo/client';
import { SSELink } from './sseLink';
import { getRealtimeToken } from '@/server/actions/realtime';

const API_URL = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3001';

let _client: ApolloClient<unknown> | null = null;

/** Singleton, delta-only Apollo client. NOT the app's data layer. */
export function getRealtimeClient(): ApolloClient<unknown> {
  if (_client) return _client;
  const link = new SSELink({
    url: `${API_URL}/api/v1/graphql`,
    // Called on each (re)connect — fetches a fresh token (15-min TTL) so
    // long-lived subscriptions survive token rotation.
    headers: async () => {
      const t = await getRealtimeToken();
      return t ? { Authorization: `Bearer ${t.token}` } : {};
    },
  });
  _client = new ApolloClient({
    link,
    cache: new InMemoryCache(),
    defaultOptions: { watchQuery: { fetchPolicy: 'no-cache' }, query: { fetchPolicy: 'no-cache' } },
  });
  return _client;
}
```

> `graphql-sse`'s `headers` option accepts an async function invoked per connection — this refreshes the 15-minute token on reconnect. `NEXT_PUBLIC_API_URL` is the browser-reachable API origin (CORS allows it with `Authorization` + credentials).

Add a mapper so the live payload matches the component's `Comment` shape (subscription returns a subset; missing fields default until the next SSR refetch):
```tsx
function mapLiveComment(c: any) {
  return {
    id: c.id, authorId: c.authorId, authorName: null,
    body: c.body, isEdited: false, createdAt: c.createdAt,
    reactions: [], assignedToId: null, resolvedAt: null,
```
